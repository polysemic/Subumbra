⚖ The Dual-Job Architecture

Modern API keys and service credentials carry immense risk. Most enterprise security tooling addresses this challenge in silos—either as a Secret Vault (which secures credentials at rest but exposes them upon request) or as an API Traffic Broker (which routes calls but requires you to hand over plaintext keys).

Subumbra is architected to perform both functions seamlessly, maintaining cryptographic isolation throughout the entire lifecycle.
▣ Job 1: Split-Trust Custody (Vault)

Traditional key storage requires storing raw master secrets in a central database. If the database or cluster is breached, every credential is lost. Subumbra implements Multi-Party Cryptography (Split-Keys) across a local Docker environment and an isolated Cloudflare Worker, meaning neither system ever possesses or records the full secret in plaintext at rest.

→ Eliminates the central "honeypot" vulnerability of traditional key vaults.
⌖ Job 2: Local Security Mediation (Broker)

Standard API proxies and routers require you to upload your master secrets directly to their platforms so they can load balance or rate limit requests. Subumbra mediates API traffic **locally at the edge**, proxying requests and dynamically signing/decrypting them via isolated blind handshakes—without the proxy ever holding your master keys.

→ Secures your keys in transit, enforcing session quarantine and forensic audit logs.
