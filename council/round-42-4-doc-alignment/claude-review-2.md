## New Finding 3 — CLAUDE.md Env Vars Section Missing *_ALLOWED_KEYS Variables

Comparing `CLAUDE.md:208-219` (runtime env vars) against `.env.example`:

**In `.env.example`, NOT in `CLAUDE.md`:**
```
PROXY_ALLOWED_KEYS=    ← generated by post-bootstrap.sh; required for proxy operation
PROBE_ALLOWED_KEYS=    ← generated by post-bootstrap.sh
UI_ALLOWED_KEYS=       ← generated by post-bootstrap.sh
TUNNEL_TOKEN=          ← optional cloudflared token
AUDIT_MAX_ROWS=10000   ← optional, has default
```

The three `*_ALLOWED_KEYS` variables are the per-adapter key scope lists,
added in Round 35 (Adapter Flexibility). They ARE generated by
`post-bootstrap.sh` and ARE required for the proxy and probe to function.
A fresh operator reading `CLAUDE.md` would not know these variables exist.

The `TUNNEL_TOKEN` and `AUDIT_MAX_ROWS` are optional and their omission is
acceptable, but the `*_ALLOWED_KEYS` omission is a real operator-confusion risk.

**Resolution**: Change F of the proposal (update env vars) should be expanded to
add `PROXY_ALLOWED_KEYS`, `PROBE_ALLOWED_KEYS`, and `UI_ALLOWED_KEYS` to the
runtime env vars section, not just remove `SUBUMBRA_TOKEN_LITELLM`.

## New Finding 5 — CLAUDE.md UI Dashboard "Key rotation trigger" Is Not Implemented

`CLAUDE.md:222-228`:
```
## UI Dashboard Features (POC)
- List of key IDs loaded (names only, never values)
- Last request time per key
- Request count per key
- Health status of subumbra-keys container
- Key rotation trigger (re-runs bootstrap for specific key)   ← NOT in ui/app.py
- Recent request log (provider, timestamp, status)
```

`grep -n "rotat" ui/app.py` returns nothing. The UI has no rotation endpoint.
This was an aspirational feature from the initial design (`CLAUDE.md` was written
before the UI was implemented). Listing it as a current feature is misleading
for operators.

**Resolution**: Remove or strike "Key rotation trigger" from the UI Dashboard
Features list. Optionally add a note that rotation is via `--rotate` CLI flag.
