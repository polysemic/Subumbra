## 3. Proposal

### Change A — Rewrite CLAUDE.md architecture section

Replace the current architecture diagram and introductory paragraph with one
that reflects the app-owned sidecar model:

```
App (LiteLLM, OpenWebUI, N8N, etc.) — app-owned install
      ↓ api_base: http://subumbra-proxy:8090/t
subumbra-proxy  (core stack, /opt/subumbra)
      ↓ fetches V2 record + proxies to Worker
subumbra-keys   (Docker-internal only)
      ↓ returns V2 envelope — useless without RSA private key
Cloudflare Worker + Durable Object
      ↓ unwraps DEK, decrypts, injects auth (~100ms)
Provider API (Anthropic, OpenAI, Groq, etc.)
```

Update the introductory paragraph to say: `subumbra-proxy` is the current
reference integration surface. The legacy callback-era LiteLLM path
(`litellm/custom_callbacks.py`) is retained as a reference artifact but is no
longer the supported integration model.

### Change D — Replace CLAUDE.md "LiteLLM Integration" section

Rename the section to "App-Owned Integration Contract" and replace the
callback-path description with the current sidecar contract:

```
### App-Owned Integration Contract

The supported integration model is the transparent sidecar route:

- App runs in its own install (e.g. /opt/litellm, /opt/openwebui)
- App points at `subumbra-proxy` at `http://subumbra-proxy:8090/t`
- `api_key` is a plain `key_id` registered in bootstrap
- `subumbra-proxy` fetches the V2 record, packages the canonical /proxy call,
  and proxies the response; the app never sees the decrypted provider key

No callback wiring, no `subumbra:key_id` prefix, no custom transport.

### Legacy Callback Integration (Superseded)

`litellm/custom_callbacks.py` is the original Round 25-era callback-based
integration. It remains in the repo as a reference artifact. New integrations
should use the sidecar contract above.
```
